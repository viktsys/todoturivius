# Desafio: Desenvolvimento de uma aplicação de lista de tarefas

## Objetivo

Desenvolver uma aplicação web de lista de tarefas utilizando as tecnologias especificadas: Python com os frameworks Django ou Flask para o backend. O candidato deverá demonstrar habilidades básicas de desenvolvimento, incluindo a capacidade de criar APIs RESTful, manipular dados em um banco de dados, e criar interfaces de usuário interativas.

## Descrição

A aplicação consistirá em uma lista de tarefas onde o usuário poderá adicionar, remover e marcar tarefas como completas. A lista de tarefas deve ser persistente, ou seja, as tarefas adicionadas devem ser armazenadas em um banco de dados e mantidas mesmo após a atualização da página.

## Requisitos do Backend (Python com Django ou Flask)

- Criar uma API RESTful para manipulação das tarefas (CRUD)
- Utilizar um banco de dados de sua preferência para armazenar as tarefas
- Implementar endpoints para:
  - Listar todas as tarefas
  - Adicionar uma nova tarefa
  - Marcar uma tarefa como completa
  - Remover uma tarefa

## Critérios de Avaliação

- **Funcionalidade**: A aplicação deve cumprir todos os requisitos especificados
- **Qualidade do código**: O código deve ser claro, bem estruturado e seguir as melhores práticas de desenvolvimento
- **Persistência de dados**: As tarefas devem ser persistidas de forma adequada no banco de dados
- **Documentação**: O código deve estar devidamente comentado e/ou documentado

## Entrega

O candidato deve disponibilizar o código-fonte da aplicação em um repositório git (GitHub, GitLab, etc.), juntamente com instruções claras sobre como executar a aplicação localmente.

## Observação

O candidato pode escolher qualquer uma das duas opções de framework (Django ou Flask) para o backend. O uso de outras bibliotecas ou ferramentas é permitido, desde que justificado. O foco principal do desafio é avaliar as habilidades do candidato em desenvolvimento utilizando as tecnologias especificadas.
